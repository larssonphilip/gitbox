Gitbox

Everyday git interface for human beings.

Gitbox is a Mac OS X graphical interface for Git version control system. In a single window you see branches, history and working directory status.

Everyday operations are easy: stage and unstage changes with a checkbox. Commit, pull, merge and push with a single click. Double-click a change to show a diff with FileMerge.app

Features

Checkout Everything. You may checkout a local branch, tag or even a remote branch with a single dropdown button. From now on you don’t have to remember obscure branch-tracking commands or edit .git/config. Select remote branch, type a local name for it and you’re done.

Always Ready. Gitbox updates working directory status each time you focus the window. When window is out of focus, it periodically fetches data from the selected remote branch and displays unmerged commits in the history. The latest status of the local folder and a server is always up to date.

Rich History. History shows all commits on both local and remote branches. Non-pushed commits are green. Non-pulled commits are grey. You can even select another local branch in a “Remote branch” menu to merge it in the current one.

Clean Design. There are no palettes or additional windows cluttering a screen. Your repository fits nicely inside a single window with few buttons. Use main menu for infrequent operations such as adding a new repository URL or stashing away changes.

Super Fast. We all love Terminal for the speed. One-letter aliases and shell scripts make everyday tasks really fast. But Gitbox is even better. Say, you move a file from one folder to another. In the Terminal you would have to “git rm” the old path (the file is gone, so no tab-completion) and “git add” the new path. Deleting multiple files is just a pain. But in Gitbox it is just a couple of checkbox clicks. And since status is updated instantly, you never have to type “git status” ever again.
